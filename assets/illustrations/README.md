# Illustration Assets

**Illustrations will be dropped here as they are delivered.**

---

## Sanctioned Style

The one sanctioned illustration system for this design standard is **[Xiaohei by Ian Neo](https://github.com/helloianneo/ian-xiaohei-illustrations)** — modernist, warm, readable. Cute without being childish. Expressive without being decorative.

This is the Sato Exception applied to imagery: permitted where warmth does measurable human work, never where decoration is the reason.

## Naming Convention

```
[context]-[subject]-[number].png
```

Examples:
- `empty-no-data-01.png`
- `error-connection-lost-01.png`
- `onboard-first-login-01.png`
- `play-card-intro-01.png`

## Context Tags

| Context | When to use |
|---|---|
| `empty-` | Empty states — dashboard with no data loaded yet |
| `error-` | Error states — connection lost, API down, data unavailable |
| `onboard-` | First contact with a complex system — orientation, welcome |
| `play-` | Play-mode artifacts — game cards, flashcards, workbooks |
| `explainer-` | Sidebar callout explaining a concept, not a data point |

## Grammar

- **One character per surface.** The Divine Move rule applies to illustration. Never two characters on the same panel.
- **Keep the character's own palette.** Do not remap to Rams tokens. The contrast between the illustration's warmth and the mono field is intentional.
- **Never full-bleed or background.** The illustration is an element, not a surface.
- **Scale:** small enough not to compete with data; large enough to be read in one glance.
- **Position:** empty states → centered; error states → left of error message; onboarding → above or beside text.

## Where It Does NOT Belong

- Inside data cells, rows, or charts
- Hero sections of live Instrument-mode dashboards (they stay cold and exact)
- Anywhere the Rams field is doing its job: the illustration is for the human moment between operations, not during them

## Format

- PNG with transparency, 2× resolution minimum
- Max display width: 200px on desktop, 160px on mobile
- Alt text required: describe what the character is doing, not what it looks like
